# 软件测试 - 日志文件上传功能（LOG-UP-01）

### 一、测试目标



1. 验证日志文件上传功能（LOG-UP-01）及所有子功能（LOG-UP-01-01\~03）符合需求规格说明，满足用户实际使用场景。

2. 确保功能在不同环境、不同操作方式下的稳定性、兼容性和易用性。

3. 识别并记录功能存在的缺陷，包括功能异常、性能瓶颈、交互体验问题等，为开发修复提供依据。

4. 验证上传过程中的数据完整性，确保文件上传后无内容丢失、格式错乱等问题。

### 二、测试范围

#### （一）功能测试范围



1. 核心功能：拖拽上传、文件选择器上传两种方式的可用性。

2. 子功能：

* LOG-UP-01-01：.log、.txt、.json 格式文件的支持情况，非支持格式文件的拦截处理。

* LOG-UP-01-02：文件大小检测逻辑，≤1GB 文件的正常上传，＞1GB 文件的拦截提示。

* LOG-UP-01-03：上传进度实时显示准确性，取消上传功能的有效性（取消后是否终止传输、释放资源）。

1. 输入验证：空文件上传、特殊命名文件（含特殊字符、超长文件名）上传的处理逻辑。

#### （二）非功能测试范围



1. 性能测试：单文件上传响应时间（≤100MB 文件≤3 秒，1GB 文件≤合理阈值）、并发上传（2-5 个用户同时上传≤500MB 文件）的稳定性。

2. 兼容性测试：支持的 VS Code 版本（最新 3 个稳定版）、Windows（10/11）、macOS（最新 2 个版本）、Linux（Ubuntu 20.04/22.04）系统兼容性。

3. 易用性测试：拖拽区域提示清晰、文件选择器操作流畅、进度条直观、取消按钮易点击。

4. 异常测试：网络中断 / 恢复、磁盘空间不足、文件被占用时的上传处理及提示。

1. 缺陷等级定义：

* 致命（Blocker）：核心功能无法使用（如无法上传任何文件），需立即修复。

* 严重（Critical）：重要功能异常（如 1GB 文件无法上传），影响主要使用场景。

* 一般（Major）：非核心功能问题（如进度条显示延迟 1 秒），不影响使用但需优化。

* 轻微（Minor）：易用性 / 界面问题（如提示文案不规范），可后续迭代修复。
